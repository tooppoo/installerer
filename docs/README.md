# installerer Documentation

This index explains who each documentation area is for and where to start. It contains navigation only; the substantive content lives in the linked documents.

## For Users

Read [the user guide](guide/README.md) if you use `installerer` to generate an installer, or if your project's GitHub Releases must satisfy what a generated installer expects. It covers the release asset contract, latest/pinned install semantics, and the runtime behavior of the generated `install.sh`.

## Generated References

The documents under [docs/reference/](reference/) are generated from the implementation. They are authoritative for the exact facts they list and must not be edited by hand; each file's header names its source of truth and regeneration command.

- [Generated Installer Runtime Dependencies](reference/runtime-dependencies.md) — the commands a generated installer requires at runtime.
- [CLI Exit Codes](reference/exit-codes.md) — the stable exit code table of the `installerer` CLI.

Regenerate with `bun run docs:generate`; CI detects drift via `bun run docs:check`.

## For Developers

Read [the design documentation](design/README.md) when changing `installerer` itself. It explains design structures that the implementation alone does not preserve, and points to where the authoritative definitions live.

## Decision Records

[docs/adr/](adr/README.md) preserves architecture decisions: the context at decision time, the alternatives considered, and the trade-offs accepted. Read an ADR when you need to know why something is the way it is; read the guide and design documents for how it currently works. ADRs are historical records and are never rewritten to match later changes.
